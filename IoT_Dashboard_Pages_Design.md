### Zone 1: AI 安心看板

**内容**:
- 早晨情境化摘要，最多 3 条信息：
  - ☀️ 早上好！家里一切正常，今天天气晴 25°C
  - 💊 提醒：降压药 7:30 该吃了
  - 🔒 门锁已锁好，放心
- 两个操作按钮：[✅ 我知道了] [💊 已吃药]

**AIOS 模块职责**:

| 模块 | 职责 | 处理细节 |
|------|------|---------|
| **Contextual** | ① 获取当前时间（早晨）+ 天气（晴 25°C）<br>② 获取用药计划：降压药 7:30<br>③ 获取全屋设备状态摘要：门锁已锁、无异常<br>④ Memory：老人通常 7:00 起床，降压药常延迟 10 分钟才吃 | - 从健康 App 获取用药计划上下文<br>- 从 SmartThings 获取设备异常状态<br>- 长期记忆：降压药经常延迟 → 提前 5 分钟提醒 |
| **Agentic** | ① 推理早晨最需要提醒的事项：用药 > 安防确认 > 天气<br>② 决定语气：温暖、安心<br>③ 判断升级策略：用药 30 分钟未确认 → 通知家人 | - 优先级：安全告警 > 用药提醒 > 健康建议 > 日常问候<br>- 早晨语气：温暖安心，"家里一切正常"给安全感 |
| **Capability** | ① 获取天气 API 数据<br>② 获取设备状态<br>③ 语音播报（TTS）"早上好，降压药该吃了" | - 调用天气 API<br>- 调用 SmartThings 设备状态 API<br>- 调用 TV 音箱 TTS |
| **Presentation** | ① 根据早晨情境动态生成看板<br>② 暖色调 + 太阳图标<br>③ 提醒项最多 3 条，避免信息过载<br>④ 每条提醒配一键确认按钮 | - 早晨配色：暖黄渐变背景<br>- 提醒卡片：白色圆角卡片 + 大字体<br>- 确认按钮：超大尺寸，遥控器一键触发 |

### Zone 2-A: 用药提醒

**内容**:
- 今日用药列表，按时间排序：
  - 🔴 降压药 7:30 ← 当前需服用（红色高亮 + 脉冲动画）
  - ⚪ 钙片 12:00 ← 待服用（灰色）
  - ⚪ 降糖药 18:00 ← 待服用（灰色）
- 大按钮：[✅ 已吃药]

**AIOS 模块职责**:

| 模块 | 职责 | 处理细节 |
|------|------|---------|
| **Contextual** | ① 获取用药计划：降压药 7:30、钙片 12:00、降糖药 18:00<br>② 获取今日用药记录：均未服<br>③ Memory：降压药常延迟 10 分钟 → 提前提醒 | - 从健康 App/日历获取用药计划<br>- 记录每次确认时间和方式<br>- 长期记忆：降压药经常延迟 → 7:25 开始提醒 |
| **Agentic** | ① 推理提醒时机：老人 7:00 起床 → 7:25 开始提醒降压药<br>② 推理确认方式：语音"吃过了"/按钮点击/摄像头自动确认<br>③ 升级策略：15 分钟后再提醒 → 30 分钟后通知家人 | - 老人今天 7:00 起床 → 7:25 开始提醒<br>- 摄像头检测到服药动作 → 自动确认<br>- 连续 2 天漏服 → 通知家人 |
| **Capability** | ① 语音播报"王奶奶，该吃降压药了"<br>② 摄像头调用与动作识别（自动确认）<br>③ 通知推送（家人，升级时）<br>④ 用药记录写入 | - 调用 TV 音箱 TTS<br>- 调用摄像头 API 动作识别<br>- 调用消息推送 API<br>- 写入健康记录数据库 |
| **Presentation** | ① 当前药物红色高亮 + 脉冲动画<br>② 待服药物灰色<br>③ 已服药物绿色 ✅ + 删除线<br>④ 确认按钮超大尺寸<br>⑤ 确认后动画：✅ 弹出 + 语音"好的，已记录" | - 当前药物：红色背景 + 白色大字<br- 待服药物：浅灰背景<br>- 确认动画：绿色 ✅ 从小到大弹出 |

### Zone 2-B: 健康速览

**内容**:
- 4 个核心健康指标，一目了然：
  - 血压 128/82 ✅ 正常
  - 步数 350 🚶 目标 3,000（早晨刚起，步数少正常）
  - 室温 24°C 😊 舒适
  - 体重 62kg ➡️ 稳定

**AIOS 模块职责**:

| 模块 | 职责 | 处理细节 |
|------|------|---------|
| **Contextual** | ① 获取健康设备数据：血压 128/82、步数 350、体重 62kg<br>② 获取室内环境：温度 24°C、湿度 55%<br>③ Memory：老人正常血压范围 120-135、体重趋势稳定 | - 从健康设备 API 获取最新数据<br>- 从 SmartThings 传感器获取环境数据<br>- 长期记忆：老人正常血压基线 |
| **Agentic** | ① 推理健康状态：血压 128/82 → 正常 ✅<br>② 推理步数：早晨 350 步 → 正常（刚起床）<br>③ 推理室温：24°C → 舒适<br>④ 异常联动：血压偏高 → 自动调暗灯光 + 建议休息 | - 血压 150/95 → 异常，结合昨晚睡眠差 → 推理原因<br>- 室温 28°C + 血压偏高 → 推理为温度导致 → 建议调低空调 |
| **Capability** | ① 获取健康设备数据<br>② 调整家居环境（异常时联动）<br>③ 通知家人（异常时） | - 调用健康设备数据 API<br>- 调用 SmartThings 空调/灯光控制<br>- 调用消息推送 API |
| **Presentation** | ① 正常指标：绿色 ✅<br>② 关注指标：黄色 ⚠️<br>③ 异常指标：红色 🔴 + 闪烁<br>④ 只展示 4 个核心指标，不堆砌数据 | - 每个指标一行，左图标 + 中数值 + 右状态<br>- 字体 28pt，远距离可读 |

### Zone 3: 一键操作区

**内容**:
- 5 个超大按钮，固定布局：
  - 💡 灯光 — 关（当前状态）
  - ❄️ 空调 — 25°C（当前温度）
  - 🔐 门锁 — 已锁（当前状态）
  - 📞 家人 — 一键拨打
  - 🆘 求助 — 红色紧急按钮

**AIOS 模块职责**:

| 按钮 | Contextual | Agentic | Capability | Presentation |
|------|-----------|---------|------------|-------------|
| **💡 灯光** | 获取灯光状态：关；Memory：老人早晨习惯开客厅灯 | 理解"开灯"→ 开客厅灯；推理位置→只开当前房间 | 调用 SmartThings 灯光 API | 按钮显示"关"；点击切换为"开" |
| **❄️ 空调** | 获取空调状态：25°C 运行中；Memory：老人舒适温度 25°C | 理解"太冷了"→ 调高 2°C；血压高 → 不建议低于 24°C | 调用 SmartThings 空调 API | 按钮显示"25°C"；点击开/关 |
| **🔐 门锁** | 获取门锁状态：已锁；Memory：老人常忘锁门 | 晚上 10 点未锁 → 主动提醒；语音"锁门"→ 执行 | 调用 SmartThings 门锁 API | 按钮显示"已锁"✅；点击切换 |
| **📞 家人** | 获取家人联系方式；Memory：常联系人排序 | "想儿子了"→ 拨打儿子电话 | 调用视频通话 API | 大头像按钮；点击直接拨打 |
| **🆘 求助** | 获取紧急联系人；Memory：老人健康风险等级 | "救命"→ 拨打 120 + 通知家人 + 开门锁；误触 → 语音确认 | 拨打紧急电话；解锁门锁；通知家人 | 红色大按钮；防误触二次确认 |
